# ADR-0004/0008/0009/0010 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement the budget-as-prediction model (ADR-0004), task creation with optional role (ADR-0008), preparation/publication functions (ADR-0009), and self-optimization governance (ADR-0010).

**Architecture:** Four coordinated changes: (1) yoitsu-contracts adds max_cost to RoleMetadata and observation event types, (2) Trenni validates max_cost at spawn time and emits observation events, (3) Palimpsest removes cost-based termination, renames workspace→preparation, (4) evo/roles get max_cost declarations.

**Tech Stack:** Python, Pydantic, dataclasses, async event handling

---

## Overview

### ADR-0004: Budget as Prediction
- Remove cost-based termination from Palimpsest runtime
- Add `max_cost` to RoleMetadata for spawn-time validation
- Track cost for observation but not enforcement
- Rename iteration penalty fields to suspended

### ADR-0008: Task Creation and Ingestion
- Allow spawn without role (default to planner)
- Add trigger evaluator module in Trenni (Phase 2, scaffold only)
- Document trigger rule schema (defer full implementation)

### ADR-0009: Preparation and Publication Functions
- Rename `workspace_fn` → `preparation_fn`, `WorkspaceConfig` → `PreparationConfig`
- Rename `setup_workspace` → `run_preparation`
- Update role definitions to use new naming
- Add preparation failure observation event

### ADR-0010: Self-Optimization Governance
- Define observation event schemas in yoitsu-contracts
- Add observation event emission in Trenni (budget_variance, preparation_failure)
- Add causal chain (task_id, job_id, role) to observation events
- Scaffold review role check items (defer full implementation to Phase 5)

```python
from __future__ import annotations

from dataclasses import dataclass


@dataclass
class ObservationBudgetVarianceData:
    """Emitted after job completion: planner's estimate vs actual spend.
    
    Per ADR-0010: budget prediction accuracy is a proxy for system modeling fidelity.
    """
    task_id: str
    job_id: str
    role: str
    estimated_budget: float
    actual_cost: float
    variance_ratio: float  # (actual - estimated) / estimated; positive = underestimate


@dataclass
class ObservationPreparationFailureData:
    """Emitted when preparation function fails.
    
    Per ADR-0009: preparation failure marks job as failed immediately.
    This event provides visibility for optimization.
    """
    task_id: str
    job_id: str
    role: str
    error_type: str
    error_message: str


@dataclass
class ObservationToolRetryData:
    """Emitted when tool call fails and is retried.
    
    Per ADR-0010: tool retry patterns indicate reliability issues.
    """
    task_id: str
    job_id: str
    role: str
    tool: str
    failure_pattern: str
    retry_count: int


# Event type namespace constants
```

```python
# Modify palimpsest/palimpsest/runtime/llm.py

# In UnifiedLLMGateway.budget_exhausted():
def budget_exhausted(self) -> str | None:
    """Check if any system backstop is exhausted.
    
    Per ADR-0004: cost is NOT an enforcement dimension.
    Only iterations_hard, input_tokens, output_tokens trigger termination.
    """
    # Check hard iteration ceiling
    if self._config.max_iterations_hard > 0 and self.total_iterations >= self._config.max_iterations_hard:
        return "max_iterations_hard"
    
    # Check token limits (system backstops)
    if self._config.max_total_input_tokens > 0 and self.total_input_tokens >= self._config.max_total_input_tokens:
        return "input_tokens"
    
    if self._config.max_total_output_tokens > 0 and self.total_output_tokens >= self._config.max_total_output_tokens:
        return "output_tokens"
    
    # NOTE: cost is intentionally NOT checked here
    # Cost tracking remains active for observation but not enforcement
    return None
```

```python
async def _emit_budget_variance(self, job_id: str, job: JobHandle, completed_event) -> None:
    """Emit budget variance observation after job completion."""
    spawn_defaults = self._spawn_defaults_by_job.get(job_id)
    if not spawn_defaults:
        return
    
    estimated_budget = spawn_defaults.budget or 0.0
    actual_cost = completed_event.data.get("cost", 0.0)  # from JobCompletedData
    
    if estimated_budget > 0:
        variance_ratio = (actual_cost - estimated_budget) / estimated_budget
        
        await self.client.emit(
            OBSERVATION_BUDGET_VARIANCE,
            ObservationBudgetVarianceData(
                task_id=job.task_id,
                job_id=job_id,
                role=job.role,
                estimated_budget=estimated_budget,
                actual_cost=actual_cost,
                variance_ratio=variance_ratio,
            ).model_dump(),
        )
```
